核心理念

Joi 是 local-first Personal Agent OS：自然聊天在前台，受控执行在后台，模型、工具、节点、记忆和交付物都要可追溯。

当前前台体验采用“伪装群聊”：用户看到私人总群，里面有真人用户和多个项目人格。群聊只是降低认知成本的 UI 隐喻，底层仍由 Orchestrator Core、tool-calling runtime、Capability policy、Memory OS 和 Run Trace 控制。

“项目作为 agent 人格”是核心表达：每个项目可拥有名称、头像、项目路径、自述、权限规则和模型策略。Agent 是岗位和职责边界，模型只是可替换的执行引擎。

主聊天保持自然消息瀑布流；右侧 inspector 承接运行、线程、资产、记忆和成员/项目详情；设置中心作为高级管理入口。
